## Regras de Correção
1. Corrija APENAS o problema indicado acima.
2. NUNCA use DELETE, DROP, UPDATE, INSERT, ALTER ou qualquer operação que modifique dados.
3. Use APENAS tabelas e colunas que existem no schema fornecido.
4. Use aliases descritivos em português para as colunas.
5. Se a pergunta for ambígua ou impossível de responder com os dados disponíveis, retorne APENAS a palavra "FORA_DO_ESCOPO" seguida de uma breve explicação.
6. Retorne APENAS o SQL corrigido, sem explicações, dentro de um bloco ```sql```.

## Exemplo
SQL Problemático:
```sql
SELECT nome, inexistente FROM clientes;
```

Erro Detectado:
Coluna 'inexistente' nao esta no allowlist do schema.

SQL Corrigido:
```sql
SELECT nome, regiao FROM clientes LIMIT 1000;
```
